# Explorer 多标签与冲突弹窗修复设计

## 问题

Windows 11 的多个 Explorer 标签会在 `Shell.Application.Windows()` 中生成多条记录，但共享同一个顶层窗口句柄。旧实现只按顶层句柄匹配并返回第一条记录，因此在第二个标签按 `Ctrl+V` 时可能粘贴到第一个标签。冲突弹窗采用固定坐标和尺寸，在高 DPI 或较大字体下会裁切文字；弹窗也没有 Explorer owner，可能落到其他窗口之后。

## 方案

对每条 Shell 记录通过 `IServiceProvider` 获取 `IShellBrowser`，调用 `QueryActiveShellView` 并取得标签独立的视图句柄。把键盘钩子捕获的焦点窗口沿父窗口链向上匹配视图句柄，以选出真正活动的标签。若同一 Explorer 窗口存在多个候选目录但无法确认活动视图，则不猜测路径，而是放回原生 `Ctrl+V`。

冲突弹窗使用表格和流式布局、可横向滚动的列表以及字体自动缩放。触发任务时保存 Explorer 顶层句柄，显示弹窗时将其作为 owner；弹窗在生命周期内设为 TopMost，并主动激活，默认焦点保持在“取消”。

## 验证

使用两个不同目录的 Explorer 标签分别运行活动目录探测，确认返回当前标签路径。使用高 DPI 冲突示例检查标题、说明、路径列表和按钮不被裁切，并确认弹窗位于 Explorer 之上。最后执行完整构建、安装回归、复制和剪切集成测试。
